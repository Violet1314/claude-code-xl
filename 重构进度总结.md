## 四、核心模块详解

### 4.1 app.py - 主应用类

**核心属性**：
- `MAX_TOOL_ROUNDS = 80`：最大工具循环轮次
- `MAX_TOOLS_PER_ROUND = 50`：每轮最大工具数
- `_last_sigint`：CTRL+C 中断时间戳
- `_plan_mode: bool`：是否处于计划模式
- `_plan_task: str`：计划模式任务描述
- `_plan_reminder_count: int`：计划模式连续提醒计数（熔断用）

**状态同步**：
- `_update_input_state()`：同步 plan_mode 到输入处理器，计划模式下 prompt 显示 `●` 前缀

**主循环流程**：
```python
def run():
    show_status_bar()           # 状态栏（模型/Token/费用）
    user_input = prompt()
    if not user_input:
        console.info("请输入内容，或使用 /help 查看帮助")  # 空输入反馈
    if user_input.startswith('/'):
        execute_command()
        console.success("命令已执行")  # 命令执行反馈
    chat(user_input)
```

**对话流程**：
```python
def chat(user_input):
    while round_count < MAX_TOOL_ROUNDS:
        messages = get_optimized_messages()  # 倒序修剪
        response, has_tools = _handle_response()
        if consecutive_failures >= 3:        # 连续错误熔断
            break
        if has_tools:
            execute_tools()
```

**中断机制**：
- 单击 CTRL+C：标记中断，流式请求/工具执行可中断
- 双击 CTRL+C（<1秒）：直接 `sys.exit(0)`

**后台计时线程**：
```python
def _refresh_timer():
    # 每0.1秒刷新进度条时间显示
    while not stop_timer_event.is_set():
        elapsed = time.time() - start_time
        progress.update(task, description=f"... {elapsed:.1f}s")
        stop_timer_event.wait(0.1)
```

**费用计算**：
```python
def _calculate_cost(input_tokens, output_tokens):
    # 费用 = (input × input_price + output × output_price) / 1,000,000
    input_price, output_price = current_model.get_prices()
    return (input_tokens * input_price + output_tokens * output_price) / 1_000_000
```

**计划模式提醒注入**（chat() 内）：
```python

if self._plan_mode:

        "2. 每开始一个任务，先调 TodoUpdate(id, \"in_progress\")\n"
        "3. 完成后调 TodoUpdate(id, \"completed\")\n"

        "6. 全部完成后，给出最终总结\n"

    conversation.add_user_message(plan_prompt)

# 2. 工具执行后刷新 Todo 面板 + 检测完成
if self._plan_mode and has_tools:
    self._plan_reminder_count = 0
    show_todo_panel(todo)
    if todo.is_all_done:
        show_plan_complete(todo)
        self._plan_mode = False

# 3. 模型未调用工具时，注入行动提醒（含熔断）
if self._plan_mode and not has_tools and not todo.is_all_done:
    self._plan_reminder_count += 1
    if self._plan_reminder_count > PLAN.REMINDER_MAX:  # 熔断
        show_plan_aborted(f"模型连续 {REMINDER_MAX} 次未按计划调用工具推进任务", todo)
        self._plan_mode = False
        break
    # 构建具体行动指令
    if in_progress_items:
        action_hint = "请立即调用 TodoUpdate(id=..., status=\"completed\") 标记当前任务为完成"
    elif pending_items:
        action_hint = "请调用 TodoUpdate(id=..., status=\"in_progress\") 开始下一个任务"
    # 连续无工具加强警告
    if self._plan_reminder_count >= PLAN.NO_TOOL_ROUNDS_MAX:
        no_tool_warning = f"⚠ 已连续 {count} 轮未调用任何工具！"
    remind = f"[系统提醒] 行动：{action_hint}（提醒 {count}/{REMINDER_MAX}）"
    conversation.add_user_message(remind)
    continue
```

## 十二、注意事项

### Windows PowerShell 兼容性

- `mkdir data, output`（多目录逗号分隔）
- `Get-ChildItem` 或 `ls`（不带 Unix 参数）
- 不支持 `mkdir -p`、`rm -rf`、`cp -r`

### Edit 精确匹配要求

必须从 Read 结果精确复制，包含前后 2-3 行上下文

- `python -i`、`vim`、`mysql` 等会被提前拦截
