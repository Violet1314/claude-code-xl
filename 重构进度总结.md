## 五、工具系统 (v2.8.2 逻辑增强)

### 5.1 工具调用模式
*   统一使用 Native Tool Calling（v2.7.0 起）。
*   通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。
*   支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统 (v2.8.2 核心优化)
*   **核心设计**：
    *   首次读取：读取文件 → 完整缓存 → 返回内容 + 引用。
    *   后续操作：AI 基于缓存回答 → 无需重新读取。
    *   **版本隔离**：写入/编辑后自动更新版本（v0 → v1 → v2...），**新版本读取计数器从 0 开始**，避免因旧版本历史导致误拦截。
*   **重复读取检测**：
    *   第 1 次：正常读取。
    *   第 2-4 次：终端显示黄色警告 `⚠️ 该文件已读取 X 次`，但允许执行。
    *   第 5 次起：**智能拦截**，返回详细错误建议（检查缓存、使用 offset/limit、确认外部修改）。
*   **缓存记录内置**：`ReadTool.execute()` 内部调用 `file_cache.record_read()` 并检查 `blocked` 状态。

### 5.3 内置工具
| 工具 | 参数 | 说明 |
| --- | --- | --- |
| Read | file_path, offset?, limit? | 读取文件（卡片式输出，集成智能拦截） |
| Write | file_path, content | 创建或覆盖文件（卡片式输出） |
| Edit | file_path, old_string, new_string | **【v2.8.2】** 精确替换（失败时提供相似行线索） |
| Glob | pattern, path? | 按文件名模式搜索 |
| Grep | pattern, path?, type? | 按内容搜索 |
| Bash | command, timeout?, cwd? | 执行 shell 命令 |
| AskUserQuestion | question, options? | 向用户询问选择 |

### 5.4 多轮执行与防死循环
*   **单轮限制**：最多执行 20 个工具调用。
*   **最大循环**：10 轮（配合熔断机制）。
*   **连续错误熔断**：连续 3 次相同工具执行错误时，强制中断循环。
*   **Edit 防死循环**：匹配失败时返回 `similar_lines` 线索，引导模型先 `Read` 再 `Edit`。

### 5.5 v2.8.2 代码优化详情
1.  **Edit 工具增强 (`edit.py`)**：
    *   新增 `_find_similar_lines` 方法：在 `old_string` 匹配失败时，搜索文件中包含关键词的相似行并返回给模型。
    *   优化错误提示：明确建议“请先调用 Read 工具获取最新内容”。
2.  **文件缓存版本隔离 (`file_cache.py`)**：
    *   引入 `version_stats` 字典：记录每个版本的 `{count, ranges}`。
    *   `apply_write` 逻辑更新：写入时初始化新版本统计，计数器重置为 0。
    *   `record_read` 返回值优化：返回 `{'count': int, 'blocked': bool}`。
3.  **Read 工具拦截集成 (`read.py`)**：
    *   在 `execute` 末尾检查 `record_read` 返回的 `blocked` 状态。
    *   若被拦截，返回包含具体解决方案的 `ToolResult(error=...)`。

### 核心架构
*   **入口点**：`__main__.py` → `app.py` 的 `Application.run()`

*   **危险命令**：`DANGEROUS_PATTERNS`，直接拦截。
