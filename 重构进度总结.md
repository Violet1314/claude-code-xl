## 四-A、权限菜单
只读工具（Read、Glob、Grep）：自动放行，无需确认。
写入/执行工具（Write、Edit、Bash）：
| 选择 | 效果 |
| --- | --- |
| Yes (once) | 允许操作（同工具同路径会话内缓存，后续自动通过） |
| No (once) | 仅本次拒绝 |
| Esc/q | 取消并中断后续执行 |
敏感操作例外：`rm`、`Remove-Item`、`sudo`、`git push` 等敏感操作不缓存权限，每次都需确认。
注意：`ALL`（全局授权）选项已彻底移除，权限级别仅保留 `ONCE` 和 `NO_ONCE`。

### 5.2 文件缓存系统
核心设计（v2.7.22 代码级彻底落地）：
```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 完整缓存 → 返回内容 + 引用                   │
│  ├── ≤1500 行直接显示完整内容                               │
│  └── >1500 行显示摘要 + 结构概览                            │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于缓存回答 → 无需重新读取                         │
│  ├── 写入/编辑后自动更新版本（v0 → v1 → v2...）✅ 已闭环     │
│  ├── 读取追踪：记录次数 + 行范围（统一在 FileCacheManager）  │
│  └── 始终基于最新缓存版本                                   │
├─────────────────────────────────────────────────────────────┤
│  重复读取检测：                                             │
│  ├── 第 1 次：正常读取                                      │
│  ├── 第 2-4 次：终端显示黄色警告，模型输出附加提示 ✅ 已可见   │
│  └── 第 5 次起：拦截，不执行读取，标记为跳过 ✅ 已生效         │
└─────────────────────────────────────────────────────────────┘
```
关键参数：
`DEFAULT_LIMIT = 1500` - 默认读取行数
`SUMMARY_THRESHOLD = 1500` - 超过此行数才显示摘要
`TERMINAL_DISPLAY_THRESHOLD = 80` - 终端显示压缩阈值

## 九、给下一个 AI 的提示
### 核心架构
入口点：`__main__.py` → `app.py` 的 `Application.run()`
版本号位置：`config/defaults.py` 中的 `VERSION`
工具调用：统一使用 Native Tool Calling

### 工具系统（v2.7.22 已稳固）
工具调用管理器：`tools/tool_calling.py`，仅支持 native 模式
文件缓存管理器：`tools/file_cache.py`，统一管理内容缓存 + 读取追踪 + 版本管理
参数验证：`executor.py` 在执行前调用 `tool.validate_parameters()`
执行中断：用户取消时立即停止后续工具执行
重复检测：`executor.py` 通过 `file_cache.get_read_count()` 检测，第 2-4 次警告，第 5 次拦截（阈值 `>=4`）
**代码质量**：全工具类已彻底清理空格断字、字典键错位、变量名断裂，API 契约严格对齐，无历史技术债。

### 路径系统（重要！）
Workplace 隔离：仅 Write/Edit 的相对路径自动解析到 `workplace/` 目录
Read/Glob/Grep 不隔离：使用 `resolve_path()`，相对路径基于当前工作目录（项目根）
绝对路径：所有工具都可访问任意位置，不受隔离限制
Bash 不隔离：Bash 工具在项目根目录执行，设计如此
路径统一解析：`executor.py` 中重复读取检测使用 `resolve_path()` 统一路径

### 安全机制
只读自动放行：`executor.py` 中 `tool.is_read_only()` 为 True 时跳过权限确认
敏感命令：`bash.py` 中 `SENSITIVE_PATTERNS`，每次需确认（不缓存权限）
危险命令：`DANGEROUS_PATTERNS`，直接拦截不显示权限菜单
拦截时机：危险命令在 `executor.py` 权限确认前检查
权限级别：只有 `ONCE` 和 `NO_ONCE`

### API 与网络
用户使用中转 API：`base_url` 是 `https://yunwu.ai/v1`
异常兜底：`client.py` 使用 `except httpx.HTTPError` 捕获所有 httpx 异常
超时配置：连接 30s，读取 180s，重试 5 次

### UI 系统
主题配置：`ui/theme.py`，包含颜色、图标、Powerline 符号
卡片式输出：Read/Grep/Glob 使用 `╭─ │ ╰` 边框
文件图标：`.py` 显示 `🐍`，`.js` 显示 `📜` 等
边框颜色：`border_subtle = "#6A6A6A"`，深色终端可见
中文宽度处理：`ui/input.py` 中 `display_width()` 函数，中文=2宽度
Rich markup 转义：使用 `rich.markup.escape()` 转义文件内容中的特殊字符
Bash 流式卡片：`progress_display.py` 的 `BashStreamingDisplay`，成功和失败都在卡片内显示，executor 不再额外调用 `show_tool_result`

### 费用计算
价格解析：`settings.py` 中 `ModelConfig.get_prices()` 返回 `(input_price, output_price)`
费用公式：`(input_tokens × input_price + output_tokens × output_price) / 1,000,000`
价格配置：`price` 字段已包含分组倍率，无需额外处理

### 测试与验证
测试命令：只跑相关模块，用 `-q` 安静模式
API 稳定性测试：`test_api_stability.py` 平时忽略
**v2.7.22 验收基线**：`93 passed` + 5 步真实交互验证全通过

## 十三、下一轮测试计划
### 背景
v2.7.22 工具系统已彻底稳固，代码无历史包袱，可直接进入核心业务模块迭代。

### 经验沉淀（v2.7.22）
1. **路径一致性是第一生产力**：跨模块比较路径时，必须统一调用解析函数后再比对。`executor` 检测层与工具内部层路径 Key 不一致是重复检测失效的根源。
2. **Bash 显示架构收敛**：Bash 的 `BashStreamingDisplay` 已接管全生命周期，`executor` 层必须无条件跳过 `show_tool_result`，否则必然导致输出撕裂。
3. **Off-by-one 陷阱**：`record_read` 在执行后调用，检测在执行前。拦截阈值 `>=4` 是数学必然，切勿凭直觉改为 `>=5`。
4. **空格是隐型杀手**：字典键、变量名、正则中的尾部空格在 IDE 中不可见，但会导致跨模块契约断裂。v2.7.22 已彻底清零。

1. **`core/client.py` 重构**：优化 HTTP 流式解析、重试退避算法、中转 API 断连自动恢复机制。
2. **`app.py` 主循环优化**：增强上下文窗口动态修剪策略、多轮工具执行状态机收敛。
