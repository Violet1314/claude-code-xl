## 五、工具系统

### 5.1 工具调用模式

**统一使用 Native Tool Calling**（v2.7.0 起）

通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统

**核心设计**：

```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 完整缓存 → 返回内容 + 引用                   │
│  ├── ≤1500 行直接显示完整内容                               │
│  └── >1500 行显示摘要 + 结构概览                            │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于缓存回答 → 无需重新读取                         │
│  ├── 编辑后自动更新版本（v0 → v1 → v2...）                   │
│  └── 始终基于最新缓存版本                                   │
└─────────────────────────────────────────────────────────────┘
```

**关键参数**：
- `DEFAULT_LIMIT = 1500` - 默认读取行数
- `SUMMARY_THRESHOLD = 1500` - 超过此行数才显示摘要

### 5.3 内置工具

| 工具 | 参数 | 说明 |
|------|------|------|
| **Read** | `file_path`, `offset?`, `limit?` | 读取文件（卡片式输出） |
| **Write** | `file_path`, `content` | 创建或覆盖文件（含语法检查） |
| **Edit** | `file_path`, `old_string`, `new_string` | 精确替换（含 diff 显示） |

| **AskUserQuestion** | `question`, `options?` | 向用户询问选择 |

### 5.4 多轮执行

- 单轮最多执行 10 个工具调用
- 最多循环 5 轮（防止死循环）
- 所有工具被跳过时停止循环

## 十、v2.7.8 重构详情（API 超时优化）

### 针对国内中转 API 优化

| 配置项 | 旧值 | 新值 | 原因 |
|--------|------|------|------|
| MAX_RETRIES | 3 | **5** | 中转服务不稳定 |
| CONNECT_TIMEOUT | 10s | **30s** | 国内→境外链路长 |
| READ_TIMEOUT | 120s | **180s** | 大文件/复杂任务预留 |
| POOL_TIMEOUT | 10s | **15s** | 高峰期排队 |

### 重试策略优化

1. **随机 jitter**：`2^attempt + random.uniform(0, 1)` 避免惊群效应
2. **429 限流处理**：读取 `Retry-After` header，默认等待 60 秒
3. **错误分类重试**：5xx 重试，4xx 不重试

## 十二、v2.7.10 重构详情（UI 微调 + 费用统计）

### 菜单中文对齐修复

**问题**：`/model` 菜单中，带中文字符的模型名称导致分隔符 `│` 位置错位

**原因**：`len()` 返回字符数（中文=1），但终端显示时中文占 2 个字符宽度

**解决方案**：`ui/input.py` 添加 `display_width()` 和 `pad_to_width()` 函数

```python
def display_width(s: str) -> int:
    """计算字符串的终端显示宽度（中文=2，英文=1）"""
    width = 0
    for ch in s:
        if '\u4e00' <= ch <= '\u9fff':
            width += 2
        else:
            width += 1
    return width
```

**效果**：
```
│ ❯ Claude Sonnet 4.6 | 官转 x4 │ 200K Context
│   Claude Opus 4.6 | 官转 x4   │ 200K Context
│   Gemini 3.1 Pro | 官转 x3    │ 2000K Context
│   Grok 4.2 | default x1       │ 100K Context   ← 正确对齐
```

## 十三、v2.7.11 重构详情（Workplace 隔离目录）

### 设计目标

**痛点**：模型写文件时可能误操作项目重要文件

**方案**：提供隔离沙箱，相对路径自动重定向到 `workplace` 目录

### 路径处理规则

| 路径类型 | 处理方式 | 说明 |
|----------|----------|------|
| 相对路径 | 重定向到 `workplace/` | 模型自主决定的路径，隔离保护 |
| 绝对路径 | 保持原样 | 用户明确指定的路径，直接执行 |

### 隔离范围

| 工具 | 是否隔离 | 理由 |
|------|----------|------|
| Write | ✅ 是 | 创建/覆盖文件，需要保护 |
| Edit | ✅ 是 | 修改文件，需要保护 |
| Read | ❌ 否 | 读取无害，允许访问项目代码 |
| Glob/Grep | ❌ 否 | 搜索无害，允许访问项目代码 |

### 核心代码

**路径解析函数**（`utils/paths.py`）：

```python
def resolve_workplace_path(path: str) -> str:
    """解析 Workplace 隔离路径"""
    if os.path.isabs(path):
        return path  # 绝对路径保持原样
    else:
        return os.path.join(WORKPLACE_DIR, path)  # 相对路径重定向
```

**配置常量**（`config/defaults.py`）：

```python
WORKPLACE_DIR = "workplace"
```

**目录创建**（`app.py`）：
