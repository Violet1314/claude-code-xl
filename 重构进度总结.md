### 四-A、权限菜单
*   **只读工具**（Read、Glob、Grep）：自动放行，无需确认。
*   **写入/执行工具**（Write、Edit、Bash）：
    | 选择 | 效果 |
    | --- | --- |
    | Yes (once) | 允许操作（同工具同路径会话内缓存，后续自动通过） |
    | No (once) | 仅本次拒绝 |
    | Esc/q | 取消并中断后续执行 |
*   **敏感操作例外**：`rm`、`Remove-Item`、`sudo`、`git push` 等敏感操作不缓存权限，每次都需确认。
*   **注意**：`ALL`（全局授权）选项已彻底移除，权限级别仅保留 `ONCE` 和 `NO_ONCE`。

## 五、工具系统 (v2.8.1 代码优化)

### 5.1 工具调用模式
*   统一使用 **Native Tool Calling**（v2.7.0 起）。
*   通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。
*   支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统
*   **核心设计**：
    *   **首次读取**：读取文件 → 完整缓存 → 返回内容 + 引用。≤1500 行直接显示完整内容；>1500 行显示摘要 + 结构概览。
    *   **后续操作**：AI 基于缓存回答 → 无需重新读取。写入/编辑后自动更新版本（v0 → v1 → v2...）。
    *   **重复读取检测**：第 1 次正常读取；第 2-4 次终端显示黄色警告；第 5 次起拦截，不执行读取。
    *   **缓存记录内置**：v2.8.1 将 `file_cache.record_read()` 移至 ReadTool.execute() 内部。

### 5.3 内置工具
| 工具 | 参数 | 说明 |
| --- | --- | --- |
| Read | file_path, offset?, limit? | 读取文件（卡片式输出，缓存记录内置） |
| Write | file_path, content | 创建或覆盖文件（v2.8.1 新增卡片式输出） |
| Edit | file_path, old_string, new_string | 精确替换（含 diff 显示，多处匹配警告） |
| Glob | pattern, path? | 按文件名模式搜索（使用公共排除目录常量） |
| Grep | pattern, path?, type? | 按内容搜索（使用公共排除目录常量） |
| Bash | command, timeout?, cwd? | 执行 shell 命令（路径隔离、Unix 转换） |
| AskUserQuestion | question, options? | 向用户询问选择（选项菜单 + 自由输入） |

### 5.4 多轮执行与防死循环
*   **单轮限制**：最多执行 20 个工具调用。
*   **最大循环**：10 轮（配合熔断机制）。
*   **连续错误熔断**：连续 3 次相同工具执行错误时，强制中断循环。
*   **所有工具被跳过时**：立即停止循环。

### 5.5 v2.8.1 代码优化详情
1.  **移除硬编码**：
    *   `executor.py`：`file_cache.record_read()` 移至 ReadTool 内部。
    *   `permission.py`：改用 `tool.get_security_context()` 获取敏感度。
2.  **公共常量抽取**：
    *   `EXCLUDED_DIRS` 从 glob.py/grep.py 抽取到 `utils/paths.py`。
3.  **统一实现模式**：
    *   Tool 基类新增 `parameters` 属性默认值。
    *   各工具 `get_security_context()` 移除 `hasattr` 检查。
4.  **Write 工具增强**：
    *   新增 `_build_terminal_display()` 方法。
    *   返回 `display_output` 卡片式终端输出。
5.  **Bug 修复**：
    *   `bash.py` 删除重复定义的 `is_read_only()` 方法。

### 核心架构
*   **入口点**：`__main__.py` → `app.py` 的 `Application.run()`
*   **版本号位置**：`config/defaults.py` 中的 `VERSION`
*   **工具调用**：统一使用 Native Tool Calling

### 工具系统（v2.8.1 已稳固）
*   **Tool 基类**：新增 `parameters` 属性默认值，`get_security_context()` 无需 `hasattr` 检查。
*   **公共常量**：`utils/paths.py` 中的 `EXCLUDED_DIRS`，Glob/Grep 统一使用。

*   **Workplace 隔离**：Write/Edit/Bash 的相对路径自动解析到 `workplace/` 目录。
*   **Read/Glob/Grep 不隔离**：使用 `resolve_path()`，相对路径基于项目根目录。

*   **只读自动放行**：`tool.is_read_only()` 为 True 时跳过权限确认。

*   **危险命令**：`DANGEROUS_PATTERNS`，直接拦截。
*   **权限级别**：只有 `ONCE` 和 `NO_ONCE`。

### UI 系统
