*   **只读工具**（Read、Glob、Grep）：自动放行，无需确认。
*   **写入/执行工具**（Write、Edit、Bash）：

## 五、工具系统 (v2.8.0 核心重构)

### 5.1 工具调用模式
*   统一使用 **Native Tool Calling**（v2.7.0 起）。
*   通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。
*   支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统
*   **核心设计**（v2.7.22 代码级彻底落地，v2.8.0 保持稳定）：
    *   **首次读取**：读取文件 → 完整缓存 → 返回内容 + 引用。≤1500 行直接显示完整内容；>1500 行显示摘要 + 结构概览。
    *   **后续操作**：AI 基于缓存回答 → 无需重新读取。写入/编辑后自动更新版本（v0 → v1 → v2...）。
    *   **重复读取检测**：第 1 次正常读取；第 2-4 次终端显示黄色警告；第 5 次起拦截，不执行读取。

### 5.3 内置工具
| 工具 | 参数 | 说明 |
| --- | --- | --- |
| Read | file_path, offset?, limit? | 读取文件（卡片式输出，终端压缩显示，缓存引用） |
| Write | file_path, content | 创建或覆盖文件（集成缓存 + 语法检查） |
| Edit | file_path, old_string, new_string | 精确替换（含 diff 显示，多处匹配警告，缓存版本递增） |
| Glob | pattern, path? | 按文件名模式搜索（卡片式输出，文件类型统计） |
| Grep | pattern, path?, type? | 按内容搜索（卡片式输出，正则支持） |
| Bash | command, timeout?, cwd? | 执行 shell 命令（默认隔离至 workplace/ ，支持 Unix 命令自动转换） |
| AskUserQuestion | question, options? | 向用户询问选择（选项菜单 + 自由输入） |

### 5.4 多轮执行与防死循环
*   **单轮限制**：最多执行 20 个工具调用（v2.8.0 提升至 20）。
*   **最大循环**：提升至 10 轮（配合熔断机制）。
*   **连续错误熔断**：检测到连续 3 次相同的工具执行错误时，强制中断循环并通知模型停止重试。
*   **所有工具被跳过时**：立即停止循环。

### 5.5 v2.8.0 架构新特性
1.  **结构化输出 (Structured Output)**：
    *   `ToolResult` 新增 `summary`、`metadata`、`display_output` 字段。
    *   工具不再手动拼接 UI 字符串，而是返回结构化数据，由 UI 层统一渲染。
2.  **安全上下文钩子 (Security Context Hook)**：
    *   `Tool` 基类新增 `get_security_context()` 方法。
    *   权限系统不再硬编码工具名，而是基于工具自述的敏感度进行决策。
3.  **执行器中间件化 (Executor Middleware)**：
    *   `executor.py` 拆分为 `_pre_execute_checks`（前置检查）、`_request_permission`（权限）、`_post_execute_handling`（后置处理）。
    *   消除了 `if tool.name == "Bash"` 等硬编码分支，提升了可扩展性。

## 九、给下一个 AI 的提示
### 核心架构
*   **入口点**：`__main__.py` → `app.py` 的 `Application.run()`
*   **版本号位置**：`config/defaults.py` 中的 `VERSION`
*   **工具调用**：统一使用 Native Tool Calling

### 工具系统（v2.8.0 已稳固）
*   **工具调用管理器**：`tools/tool_calling.py`，仅支持 native 模式
*   **文件缓存管理器**：`tools/file_cache.py`，统一管理内容缓存 + 读取追踪 + 版本管理
*   **参数验证**：`executor.py` 在执行前调用 `tool.validate_parameters()`
*   **执行中断**：用户取消时立即停止后续工具执行
*   **重复检测**：`executor.py` 通过 `file_cache.get_read_count()` 检测，第 2-4 次警告，第 5 次拦截（阈值 `>=4`）
*   **代码质量**：全工具类已彻底清理空格断字、字典键错位、变量名断裂，API 契约严格对齐，无历史技术债。
*   **架构新特性**：
    *   `ToolResult` 包含 `summary` 和 `metadata`。
    *   所有工具必须实现 `get_security_context()`。
    *   `executor.py` 使用中间件模式处理前置/后置逻辑。

### 路径系统（重要！v2.7.24 更新，v2.8.0 保持）
*   **Workplace 隔离**：Write/Edit/Bash 的相对路径自动解析到 `workplace/` 目录。
*   **Bash 特殊逻辑**：
    *   默认 `cwd` 强制指向 `workplace/` 。
    *   内置 `_translate_unix_command` 方法，自动将 `ls -la` 等转换为 `Get-ChildItem -Force` 。
    *   模型提示词已强化：“当你执行 Bash 时，你的脚下就是 workplace 目录”。
*   **Read/Glob/Grep 不隔离**：使用 `resolve_path()` ，相对路径基于当前工作目录（项目根）。
*   **绝对路径**：所有工具都可访问任意位置，不受隔离限制。
*   **路径统一解析**： `executor.py` 中重复读取检测使用 `resolve_path()` 统一路径。

### 安全机制
*   **只读自动放行**： `executor.py` 中 `tool.is_read_only()` 为 True 时跳过权限确认。
*   **敏感命令**： `bash.py` 中 `SENSITIVE_PATTERNS` ，每次需确认（不缓存权限）。
*   **危险命令**： `DANGEROUS_PATTERNS` ，直接拦截不显示权限菜单。
*   **拦截时机**：危险命令在 `executor.py` 权限确认前检查。
*   **权限级别**：只有 `ONCE` 和 `NO_ONCE` 。

### API 与网络
*   用户使用中转 API：`base_url` 是 `https://yunwu.ai/v1`
*   异常兜底：`client.py` 使用 `except httpx.HTTPError` 捕获所有 httpx 异常
*   静默重试：前 N-1 次重试不打印任何日志，仅在最终失败时显示一次错误。
*   超时配置：连接 30s，读取 180s，重试 5 次。

### UI 系统 (v2.7.25 优化后)
*   **主题配置**： `ui/theme.py` ，包含颜色、图标、Prompt Toolkit 样式。
*   **卡片体系**：全面使用 Rich `Panel` (Box: ROUNDED) 替代手动边框。
*   **文件图标**： `.py` 显示 `🐍` ， `.js` 显示 `📜` 等。
