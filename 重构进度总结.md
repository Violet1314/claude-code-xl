## 四-A、权限菜单

**只读工具（Read、Glob、Grep）**：自动放行，无需确认。

**写入/执行工具（Write、Edit、Bash）**：

| 选择 | 效果 |
|------|------|
| **Yes (once)** | 允许操作（同工具同路径会话内缓存，后续自动通过） |
| **No (once)** | 仅本次拒绝 |
| **Esc/q** | 取消并中断后续执行 |

**敏感操作例外**：rm、Remove-Item、sudo、git push 等敏感操作不缓存权限，每次都需确认。

**注意**：v2.7.21 已移除 `ALL`（全局授权）选项，权限级别只有 `ONCE` 和 `NO_ONCE`。

### 5.1 工具调用模式

通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统

**核心设计**（v2.7.21 统一管理）：

```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 完整缓存 → 返回内容 + 引用                   │
│  ├── ≤1500 行直接显示完整内容                               │
│  └── >1500 行显示摘要 + 结构概览                            │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于缓存回答 → 无需重新读取                         │
│  ├── 写入/编辑后自动更新版本（v0 → v1 → v2...）              │
│  ├── 读取追踪：记录次数 + 行范围（统一在 FileCacheManager）  │
│  └── 始终基于最新缓存版本                                   │
├─────────────────────────────────────────────────────────────┤
│  重复读取检测：                                             │
│  ├── 第 1 次：正常读取                                      │
│  ├── 第 2-4 次：终端显示黄色警告，模型输出附加提示            │
│  └── 第 5 次起：拦截，不执行读取，标记为跳过                  │
└─────────────────────────────────────────────────────────────┘
```

**关键参数**：
- `DEFAULT_LIMIT = 1500` - 默认读取行数
- `SUMMARY_THRESHOLD = 1500` - 超过此行数才显示摘要
- `TERMINAL_DISPLAY_THRESHOLD = 80` - 终端显示压缩阈值

### 5.3 内置工具

| 工具 | 参数 | 说明 |
|------|------|------|
| **Read** | `file_path`, `offset?`, `limit?` | 读取文件（卡片式输出，终端压缩显示，缓存引用） |
| **Write** | `file_path`, `content` | 创建或覆盖文件（集成缓存 + 语法检查） |
| **Edit** | `file_path`, `old_string`, `new_string` | 精确替换（含 diff 显示，多处匹配警告） |
| **Glob** | `pattern`, `path?` | 按文件名模式搜索（卡片式输出，文件类型统计） |
| **Grep** | `pattern`, `path?`, `type?` | 按内容搜索（卡片式输出，正则支持） |
| **Bash** | `command`, `timeout?`, `cwd?` | 执行 shell 命令（流式输出，敏感/危险命令检测） |
| **AskUserQuestion** | `question`, `options?` | 向用户询问选择（选项菜单 + 自由输入） |

### 5.4 多轮执行

- 单轮最多执行 20 个工具调用
- 最多循环 5 轮（防止死循环）
- 所有工具被跳过时停止循环

## 九、给下一个 AI 的提示

### 核心架构

1. **入口点**：`__main__.py` → `app.py` 的 `Application.run()`
2. **版本号位置**：`config/defaults.py` 中的 `VERSION = "2.7.21"`
3. **工具调用**：统一使用 Native Tool Calling

### 工具系统

4. **工具调用管理器**：`tools/tool_calling.py`，仅支持 native 模式
5. **文件缓存管理器**：`tools/file_cache.py`，统一管理内容缓存 + 读取追踪 + 版本管理
6. **参数验证**：`executor.py` 在执行前调用 `tool.validate_parameters()`
7. **执行中断**：用户取消时立即停止后续工具执行
8. **重复检测**：`executor.py` 通过 `file_cache.get_read_count()` 检测，第 2-4 次警告，第 5 次拦截
9. **ReadCache 已删除**：v2.7.21 将读取追踪合并到 `FileCacheManager`，executor 不再有独立的 ReadCache

### 路径系统（重要！）

10. **Workplace 隔离**：仅 Write/Edit 的相对路径自动解析到 `workplace/` 目录
11. **Read/Glob/Grep 不隔离**：使用 `resolve_path()`，相对路径基于当前工作目录（项目根）
12. **绝对路径**：所有工具都可访问任意位置，不受隔离限制
13. **Bash 不隔离**：Bash 工具在项目根目录执行，设计如此
14. **路径统一解析**：executor.py 中重复读取检测使用 `resolve_path()` 统一路径
15. **Glob/Grep 排除目录**：自动排除 `.venv`、`node_modules`、`__pycache__`、`.git` 等无关目录

### 安全机制

16. **只读自动放行**：`executor.py` 中 `tool.is_read_only()` 为 True 时跳过权限确认
17. **敏感命令**：`bash.py` 中 `SENSITIVE_PATTERNS`，每次需确认（不缓存权限）
18. **危险命令**：`DANGEROUS_PATTERNS`，直接拦截不显示权限菜单
19. **拦截时机**：危险命令在 `executor.py` 权限确认前检查
20. **权限级别**：只有 `ONCE` 和 `NO_ONCE`，v2.7.21 已移除 `ALL`

### API 与网络

18. **用户使用中转 API**：`base_url` 是 `https://yunwu.ai/v1`
19. **异常兜底**：`client.py` 使用 `except httpx.HTTPError` 捕获所有 httpx 异常
20. **超时配置**：连接 30s，读取 180s，重试 5 次

### UI 系统

21. **主题配置**：`ui/theme.py`，包含颜色、图标、Powerline 符号
22. **卡片式输出**：Read/Grep/Glob 使用 `╭─ │ ╰` 边框
23. **文件图标**：`.py` 显示 `🐍`，`.js` 显示 `📜` 等
24. **边框颜色**：`border_subtle = "#6A6A6A"`，深色终端可见
25. **中文宽度处理**：`ui/input.py` 中 `display_width()` 函数，中文=2宽度
26. **Rich markup 转义**：使用 `rich.markup.escape()` 转义文件内容中的特殊字符
27. **Bash 流式卡片**：`progress_display.py` 的 `BashStreamingDisplay`，成功和失败都在卡片内显示，executor 不再额外调用 `show_tool_result`

### 费用计算

28. **价格解析**：`settings.py` 中 `ModelConfig.get_prices()` 返回 `(input_price, output_price)`
29. **费用公式**：`(input_tokens × input_price + output_tokens × output_price) / 1,000,000`
30. **价格配置**：`price` 字段已包含分组倍率，无需额外处理

### 测试与验证

31. **测试命令**：只跑相关模块，用 `-q` 安静模式
32. **API 稳定性测试**：`test_api_stability.py` 平时忽略
33. **后续重构**：涉及测试的让用户自行测试，给出命令和路径即可
34. **test_permission.py 已同步**：v2.7.21 ALL 清理后，测试用例也已更新，不再引用 `PermissionLevel.ALL`

## 十三、下一轮测试计划

### 背景

v2.7.21 重构验收已全部通过。用户计划在下一个会话中继续测试工具类的实际使用场景。

### 本轮验收获得的经验和注意事项

1. **路径一致性是关键坑点**：workplace 隔离机制下，工具内部用 `resolve_workplace_path` 解析路径，但 executor 层面的检测如果用原始参数就会不匹配。**凡是跨模块比较路径，必须统一解析后再比较**。

2. **Bash 的显示架构**：Bash 有独立的流式卡片（`BashStreamingDisplay`），成功和失败的输出都在卡片内完成。executor 层面对 Bash 一律跳过 `show_tool_result`，否则会重复显示。

3. **测试用例要同步**：代码层面删除了 `PermissionLevel.ALL`，但 test_permission.py 还在引用，导致 3 个测试失败。**修改代码后必须检查对应测试文件**。

4. **重复读取的阈值注意 off-by-one**：`record_read` 在工具执行后调用（`_record_execution`），但重复检测在执行前。所以检测时 count=4 实际是第 5 次读取，拦截条件应为 `>= 4` 而不是 `>= 5`。

5. **终端显示 vs 模型输出**：`result.output` 是给模型的，`result.display_output` 是终端显示的。如果只往 output 追加警告，终端看不到。需要额外通过 `ui_console.print` 输出。

6. **AI 使用工具的路径行为**：AI（DeepSeek V3.2）倾向于先用相对路径，失败后才用绝对路径。workplace 隔离意味着项目根目录的文件用相对路径读不到，需要绝对路径或先复制到 workplace。
