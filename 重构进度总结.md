## 五、工具系统

### 5.1 工具调用模式

**统一使用 Native Tool Calling**（v2.7.0 起）

通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统

**核心设计**：

```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 完整缓存 → 返回内容 + 引用                   │
│  ├── ≤1500 行直接显示完整内容                               │
│  └── >1500 行显示摘要 + 结构概览                            │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于缓存回答 → 无需重新读取                         │
│  ├── 编辑后自动更新版本（v0 → v1 → v2...）                   │
│  └── 始终基于最新缓存版本                                   │
└─────────────────────────────────────────────────────────────┘
```

**关键参数**：
- `DEFAULT_LIMIT = 1500` - 默认读取行数
- `SUMMARY_THRESHOLD = 1500` - 超过此行数才显示摘要

### 5.3 内置工具

| 工具 | 参数 | 说明 |
|------|------|------|
| **Read** | `file_path`, `offset?`, `limit?` | 读取文件（≤1500行完整显示） |
| **Write** | `file_path`, `content` | 创建或覆盖文件（含语法检查+参数验证） |
| **Edit** | `file_path`, `old_string`, `new_string` | 精确替换（含 diff 显示+参数验证） |
| **Glob** | `pattern`, `path?` | 按文件名模式搜索（含参数验证） |
| **Grep** | `pattern`, `path?`, `type?` | 按内容正则搜索（含正则验证） |
| **Bash** | `command`, `timeout?`, `cwd?` | 执行 shell 命令 |

### 5.4 参数验证机制

所有工具在执行前都会验证参数：

| 工具 | 验证内容 |
|------|----------|
| Write | file_path 存在、content 不为空、路径合法 |
| Edit | file_path 存在、old_string 不为空 |
| Glob | pattern 不为空 |
| Grep | pattern 不为空、正则表达式合法 |
| Read | file_path 存在 |
| Bash | command 不为空 |

### 5.5 Edit diff 显示

编辑操作会显示 git 风格的 diff：

```
Update(test.py)
  +2 -3

     1   Line 1
     2   Line 2
     3  - def old_func():     ← 红色背景
     4  -     pass
     5  + def new_func():     ← 绿色背景
     6  +     return True
     7   Line 7
```

### 5.6 权限确认机制

所有工具操作都需要用户授权：
- **Yes (once)**: 仅本次允许
- **Yes (all)**: 授予所有工具权限（全局授权）
- **No (once)**: 仅本次拒绝，继续执行后续工具
- 按 `Esc` 或 `q` 取消操作并中断后续执行
- 长参数显示截断（content 只显示字符数）
- 敏感操作（rm、sudo、git push 等）每次都需确认

### 5.7 多轮执行

- 单轮最多执行 10 个工具调用
- 最多循环 5 轮（防止死循环）
- 所有工具被跳过时停止循环

## 九、给下一个 AI 的提示

### 核心架构

1. **入口点**：`__main__.py` → `app.py` 的 `Application.run()`
2. **版本号位置**：`config/defaults.py` 中的 `VERSION = "2.7.6"`
3. **工具调用**：统一使用 Native Tool Calling

### 工具系统

4. **工具调用管理器**：`tools/tool_calling.py`，仅支持 native 模式
5. **文件缓存管理器**：`tools/file_cache.py`，实现增量存储和版本追踪
6. **参数验证**：`executor.py` 在执行前调用 `tool.validate_parameters()`
7. **执行中断**：用户取消时立即停止后续工具执行

### API 与网络

8. **用户使用中转 API**：`base_url` 是 `https://yunwu.ai/v1`
9. **异常兜底**：`client.py` 使用 `except httpx.HTTPError` 捕获所有 httpx 异常

### 文件缓存设计

10. **首次读取完整缓存**：文件完整存入 `base_content`，后续通过 `patches` 增量更新
11. **版本追踪**：每次编辑 `version + 1`，引用格式 `[file:xxx.py:v1]`
12. **≤1500 行直接显示**：小文件不显示摘要，减少模型重复读取

### PowerShell 兼容性设计

13. **不使用静默转换**：删除了 `_convert_unix_to_powershell()` 方法
14. **语法检测反馈**：`_check_unix_syntax()` 检测不兼容语法，返回正确提示让模型自纠正
15. **环境信息注入**：`_get_environment_info()` 动态检测 OS，注入 PowerShell 语法说明
16. **执行原则注入**：system prompt 包含"持续尝试直到完成"指令

### 测试与验证

17. **测试命令**：只跑相关模块，用 `-q` 安静模式
    - 示例：`python -m pytest tests/test_permission.py -q`
    - 只看最后的 `passed` / `failed` 汇总行，避免大量输出占用上下文
18. **API 稳定性测试**：`test_api_stability.py` 调用真实中转 API，消耗 token 和时间
    - 平时重构**忽略**：`--ignore=tests/test_api_stability.py`
    - 只在改动 `core/client.py` 或 API 相关代码时才跑
    - 手动单独运行：`python tests/test_api_stability.py`
19. **测试覆盖**：核心模块已有测试（104 个）

### 重构注意事项

20. **测试先行**：重构前先跑测试确认通过
21. **已移除模块**：`tools/parser.py` 已删除，不再支持 XML/Kimi 模式
22. **参数验证位置**：在 `executor.execute_single()` 中调用，执行前验证
23. **避免大量输出**：测试时用 `-q` 模式，只看汇总结果，避免大量调试信息占用上下文导致 API 报错

## 十二、v2.7.6 重构详情

### 设计决策：放弃静默转换

v2.7.5 曾实现 `mkdir -p` → `mkdir dir1, dir2` 的静默转换，但这是**临时方案**：
- 无法覆盖所有 Unix 命令
- 模型不知道自己用了错误语法
- 不具备通用性

v2.7.6 采用**三层防御**策略：

| 层面 | 方案 | 作用 |
|------|------|------|
| Tool description | 正反示例驱动 | 告诉模型正确/错误语法 |
| System prompt | 环境信息注入 | 动态检测 OS，注入 PowerShell 语法说明 |
| 错误反馈 | `_check_unix_syntax()` | 检测不兼容语法，返回正确提示让模型自纠正 |

### 代码变更

| 文件 | 变更 |
|------|------|
| `builtins/bash.py` | 删除 `_convert_unix_to_powershell()` 静默转换方法 |
| `builtins/bash.py` | 新增 `_check_unix_syntax()` 检测 13 种 Unix 不兼容语法 |
| `builtins/bash.py` | 更新 description，加入正反示例 |
| `builtins/grep.py` | 移除 `"default": None` 避免 JSON null 问题 |
| `app.py` | 新增 `_get_environment_info()` 动态注入环境信息 |
| `app.py` | `_setup_system_prompt()` 调用环境信息注入 |
| `app.py` | `MAX_TOOL_ROUNDS` 从 3 改为 5 |

### 检测的 Unix 不兼容语法

| Unix 语法 | 错误提示 |
|-----------|----------|
| `mkdir -p` | PowerShell 不支持 -p，用 `mkdir dir1, dir2` |
| `ls -la` | PowerShell 的 ls 不支持 -l/-a，用 `Get-ChildItem` |
| `rm -rf` | PowerShell 的 rm 不支持 -r/-f，用 `Remove-Item -Recurse -Force` |
| `cp -r` | PowerShell 的 cp 不支持 -r，用 `Copy-Item -Recurse` |
| `touch` | PowerShell 没有 touch，用 `New-Item -Type File -Path name -Force` |
| `which` | PowerShell 没有 which，用 `Get-Command` |
| `find` | PowerShell 没有 Unix find，用 `Get-ChildItem -Recurse -Filter` |
| `grep` | PowerShell 没有 grep，用 `Select-String` |
| `chmod` | PowerShell 没有 chmod，用 `icacls` 或 `Set-Acl` |
| `chown` | PowerShell 没有 chown，用 `icacls` 或 `Set-Acl` |
| `cat -n` | PowerShell 的 cat 不支持 -n，用 `Get-Content` |

### 注入的 System Prompt 示例

```
## 环境信息
当前环境：Windows PowerShell。
重要：必须使用 PowerShell 语法，不支持 Unix 参数。
正确示例：
  - mkdir data, output（多个目录用逗号分隔）
  - Get-ChildItem 或 ls（不带 -la 参数）
  - Remove-Item -Recurse -Force path
  - Copy-Item -Recurse src dst
错误示例（不支持）：
  - mkdir -p data output
  - ls -la
  - rm -rf path
  - cp -r src dst

## 执行原则
遇到错误时，根据报错信息自行分析原因并修复，持续尝试直到任务完成。
不要中途放弃或等待用户干预，除非遇到无法解决的问题（如权限不足、路径不存在等）。
```
