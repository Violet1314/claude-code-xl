## 四-A、权限菜单

| 选择 | 效果 |
|------|------|
| **Yes (once)** | 仅允许当前操作，后续仍需确认 |
| **Yes (all)** | 全局授权开启，后续所有非敏感操作自动通过 |
| **No (once)** | 拒绝当前操作，继续执行后续工具 |
| **Esc/q** | 取消并中断后续执行 |

**敏感操作例外**：rm、Remove-Item、sudo、git push 等即使全局授权也需每次确认。

## 五、工具系统

### 5.1 工具调用模式

**统一使用 Native Tool Calling**（v2.7.0 起）

通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统

**核心设计**：

```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 完整缓存 → 返回内容 + 引用                   │
│  ├── ≤1500 行直接显示完整内容                               │
│  └── >1500 行显示摘要 + 结构概览                            │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于缓存回答 → 无需重新读取                         │
│  ├── 编辑后自动更新版本（v0 → v1 → v2...）                   │
│  └── 始终基于最新缓存版本                                   │
└─────────────────────────────────────────────────────────────┘
```

**关键参数**：
- `DEFAULT_LIMIT = 1500` - 默认读取行数
- `SUMMARY_THRESHOLD = 1500` - 超过此行数才显示摘要
- `TERMINAL_DISPLAY_THRESHOLD = 80` - 终端显示压缩阈值

### 5.3 内置工具

| 工具 | 参数 | 说明 |
|------|------|------|
| **Read** | `file_path`, `offset?`, `limit?` | 读取文件（卡片式输出，终端压缩显示，缓存引用） |
| **Write** | `file_path`, `content` | 创建或覆盖文件（含语法检查） |
| **Edit** | `file_path`, `old_string`, `new_string` | 精确替换（含 diff 显示，多处匹配警告） |
| **Glob** | `pattern`, `path?` | 按文件名搜索（卡片式输出，文件类型统计） |
| **Grep** | `pattern`, `path?`, `type?` | 按内容搜索（卡片式输出，正则支持） |
| **Bash** | `command`, `timeout?`, `cwd?` | 执行 shell 命令（流式输出，敏感/危险命令检测） |
| **AskUserQuestion** | `question`, `options?` | 向用户询问选择（选项菜单 + 自由输入） |

### 5.4 多轮执行

- 单轮最多执行 10 个工具调用
- 最多循环 5 轮（防止死循环）
- 所有工具被跳过时停止循环

## 九、给下一个 AI 的提示

### 核心架构

1. **入口点**：`__main__.py` → `app.py` 的 `Application.run()`
2. **版本号位置**：`config/defaults.py` 中的 `VERSION = "2.7.20"`
3. **工具调用**：统一使用 Native Tool Calling

### 工具系统

4. **工具调用管理器**：`tools/tool_calling.py`，仅支持 native 模式
5. **文件缓存管理器**：`tools/file_cache.py`，实现增量存储和版本追踪
6. **参数验证**：`executor.py` 在执行前调用 `tool.validate_parameters()`
7. **执行中断**：用户取消时立即停止后续工具执行
8. **重复检测**：`executor.py` 中第 2-4 次读取警告，第 5 次拦截

### 安全机制

9. **敏感命令**：`bash.py` 中 `SENSITIVE_PATTERNS`，每次需确认
10. **危险命令**：`DANGEROUS_PATTERNS`，直接拦截不显示权限菜单
11. **拦截时机**：危险命令在 `executor.py` 权限确认前检查

### API 与网络

12. **用户使用中转 API**：`base_url` 是 `https://yunwu.ai/v1`
13. **异常兜底**：`client.py` 使用 `except httpx.HTTPError` 捕获所有 httpx 异常
14. **超时配置**：连接 30s，读取 180s，重试 5 次

### UI 系统

15. **主题配置**：`ui/theme.py`，包含颜色、图标、Powerline 符号
16. **卡片式输出**：Read/Grep/Glob 使用 `╭─ │ ╰` 边框
17. **文件图标**：`.py` 显示 `🐍`，`.js` 显示 `📜` 等
18. **边框颜色**：`border_subtle = "#6A6A6A"`，深色终端可见
19. **中文宽度处理**：`ui/input.py` 中 `display_width()` 函数，中文=2宽度
20. **Rich markup 转义**：方括号 `[` 需转义为 `\[`，避免被解析为标签

### 费用计算

21. **价格解析**：`settings.py` 中 `ModelConfig.get_prices()` 返回 `(input_price, output_price)`
22. **费用公式**：`(input_tokens × input_price + output_tokens × output_price) / 1,000,000`
23. **价格配置**：`price` 字段已包含分组倍率，无需额外处理

### 测试与验证

24. **测试命令**：只跑相关模块，用 `-q` 安静模式
25. **API 稳定性测试**：`test_api_stability.py` 平时忽略
26. **后续重构**：涉及测试的让用户自行测试，给出命令和路径即可
