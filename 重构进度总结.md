## 四-A、权限菜单（v2.7.3）

| 选择 | 效果 |
|------|------|
| **Yes (once)** | 仅允许当前操作，后续仍需确认 |
| **Yes (all)** | 全局授权开启，后续所有非敏感操作自动通过 |
| **No (once)** | 拒绝当前操作，继续执行后续工具 |
| **Esc/q** | 取消并中断后续执行 |

**敏感操作例外**：rm、sudo、git push 等即使全局授权也需每次确认。

## 五、工具系统

### 5.1 工具调用模式

**统一使用 Native Tool Calling**（v2.7.0 起）

通过 API 原生支持实现工具调用，使用 OpenAI 格式的 `tools` 参数。支持的模型：GPT、Claude、DeepSeek、Qwen、Gemini、GLM 等主流模型。

### 5.2 文件缓存系统

**核心设计**：

```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 完整缓存 → 返回内容 + 引用                   │
│  ├── ≤1500 行直接显示完整内容                               │
│  └── >1500 行显示摘要 + 结构概览                            │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于缓存回答 → 无需重新读取                         │
│  ├── 编辑后自动更新版本（v0 → v1 → v2...）                   │
│  └── 始终基于最新缓存版本                                   │
└─────────────────────────────────────────────────────────────┘
```

**关键参数**：
- `DEFAULT_LIMIT = 1500` - 默认读取行数
- `SUMMARY_THRESHOLD = 1500` - 超过此行数才显示摘要

### 5.3 内置工具

| 工具 | 参数 | 说明 |
|------|------|------|
| **Read** | `file_path`, `offset?`, `limit?` | 读取文件（卡片式输出） |
| **Write** | `file_path`, `content` | 创建或覆盖文件（含语法检查） |
| **Edit** | `file_path`, `old_string`, `new_string` | 精确替换（含 diff 显示） |
| **Glob** | `pattern`, `path?` | 按文件名搜索（卡片式输出） |
| **Grep** | `pattern`, `path?`, `type?` | 按内容搜索（卡片式输出） |
| **Bash** | `command`, `timeout?`, `cwd?` | 执行 shell 命令（流式输出） |
| **AskUserQuestion** | `question`, `options?` | 向用户询问选择 |

### 5.4 多轮执行

- 单轮最多执行 10 个工具调用
- 最多循环 5 轮（防止死循环）
- 所有工具被跳过时停止循环

## 九、给下一个 AI 的提示

### 核心架构

1. **入口点**：`__main__.py` → `app.py` 的 `Application.run()`
2. **版本号位置**：`config/defaults.py` 中的 `VERSION = "2.7.9"`
3. **工具调用**：统一使用 Native Tool Calling

### 工具系统

4. **工具调用管理器**：`tools/tool_calling.py`，仅支持 native 模式
5. **文件缓存管理器**：`tools/file_cache.py`，实现增量存储和版本追踪
6. **参数验证**：`executor.py` 在执行前调用 `tool.validate_parameters()`
7. **执行中断**：用户取消时立即停止后续工具执行

### API 与网络

8. **用户使用中转 API**：`base_url` 是 `https://yunwu.ai/v1`
9. **异常兜底**：`client.py` 使用 `except httpx.HTTPError` 捕获所有 httpx 异常
10. **超时配置**：连接 30s，读取 180s，重试 5 次（v2.7.8 优化）

### UI 系统

11. **主题配置**：`ui/theme.py`，包含颜色、图标、Powerline 符号
12. **卡片式输出**：Read/Grep/Glob 使用 `╭─ ╭─ ╰` 边框
13. **文件图标**：`.py` 显示 `🐍`，`.js` 显示 `📜` 等
14. **边框颜色**：`border_subtle = "#6A6A6A"`，深色终端可见

### 测试与验证

## 十、v2.7.8 重构详情（API 超时优化）

### 针对国内中转 API 优化

| 配置项 | 旧值 | 新值 | 原因 |
|--------|------|------|------|
| MAX_RETRIES | 3 | **5** | 中转服务不稳定 |
| CONNECT_TIMEOUT | 10s | **30s** | 国内→境外链路长 |
| READ_TIMEOUT | 120s | **180s** | 大文件/复杂任务预留 |
| POOL_TIMEOUT | 10s | **15s** | 高峰期排队 |

### 重试策略优化

1. **随机 jitter**：`2^attempt + random.uniform(0, 1)` 避免惊群效应
2. **429 限流处理**：读取 `Retry-After` header，默认等待 60 秒
3. **错误分类重试**：5xx 重试，4xx 不重试
