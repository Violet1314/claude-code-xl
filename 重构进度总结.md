## 五、工具系统

### 5.1 工具调用模式（v2.5.0）

支持三种工具调用模式，兼容云雾 API 等中转平台：

| 模式 | 说明 | 支持模型 |
|------|------|----------|
| `native` | 使用 API 原生 tool calling | GPT, Claude, DeepSeek, Qwen, Gemini 等 |
| `xml` | 在 prompt 中定义 XML 格式 | 不支持原生调用的模型 |
| `kimi` | 解析 KIMI 特殊格式 | Kimi K2.5 |

### 5.2 文件缓存系统（v2.6.0）

**核心思想**：文件内容只完整传输一次，后续使用引用，大幅减少 Token 消耗。

```
┌─────────────────────────────────────────────────────────────┐
│                    文件缓存工作流程                          │
├─────────────────────────────────────────────────────────────┤
│  首次读取：                                                 │
│  ├── 读取文件 → 存入缓存 → 返回内容 + 引用                   │
│  └── 显示: (新缓存) 📌 引用: [file:app.py:v0]              │
├─────────────────────────────────────────────────────────────┤
│  后续操作：                                                 │
│  ├── AI 基于上下文回答 → 无需重新读取                        │
│  └── 编辑后自动更新缓存版本                                  │
└─────────────────────────────────────────────────────────────┘
```

**Token 节省效果**：

| 场景 | 之前 | 现在 | 节省 |
|------|------|------|------|
| 读取 686 行文件 | 每次完整返回 | 首次完整 + 后续引用 | ~70% |
| 追问同一文件 | 重复 4 次 Read | 0 次调用 | ~75% |
| 编辑后追问 | 重新读取 | 使用缓存引用 | ~60% |

### 5.3 内置工具

| 工具 | 参数 | 说明 |
|------|------|------|
| **Read** | `file_path`, `offset?`, `limit?` | 读取文件（集成缓存，limit=1000） |
| **Write** | `file_path`, `content` | 创建或覆盖文件 |
| **Edit** | `file_path`, `old_string`, `new_string` | 精确替换（自动更新缓存） |
| **Glob** | `pattern`, `path?` | 按文件名模式搜索（最多 100 个结果） |
| **Grep** | `pattern`, `path?`, `type?` | 按内容正则搜索（最多 30 个匹配） |
| **Bash** | `command`, `timeout?`, `cwd?` | 执行 shell 命令 |

### 5.4 权限确认机制

所有工具操作都需要用户授权：
- **Yes (once)**: 仅本次允许
- **Yes (always)**: 本次会话总是允许（缓存权限）
- 按 `Esc` 或 `q` 取消操作
- 敏感操作（如删除命令）不缓存，每次需确认

### 5.5 多轮执行

- 单轮最多执行 10 个工具调用
- 最多循环 5 轮（防止死循环）
- 工具结果自动反馈给 AI 继续处理
